\subsection{性能差异解释}

从方法机制看，性能提升可按以下链条理解：

\begin{enumerate}
    \item \textbf{RNA-only} 提供单组学可分性基线；
    \item \textbf{Concat} 利用 RNA 与甲基化互补信息，通常优于单模态；
    \item \textbf{MOFA} 通过潜因子压缩噪声与冗余，在小样本高维场景下可提升稳健性；
    \item \textbf{Stacking} 在分支层建模后再做元融合，进一步优化决策边界与泛化表现。
\end{enumerate}

结合表~\ref{tab:permtest-main} 可见，当前四方法的差异尚未达到统计显著。我们将其解释为：在小样本任务中，方法排序可作为“方向性证据”，但仍需更大样本或外部队列验证。

从主线排序看，Concat 在三项核心指标上均为第一，这与“RNA 表达与甲基化调控具有互补信息”的机制预期一致；同时，Stacking 并未稳定超过 Concat，提示在当前样本规模下，增加融合层级并不一定转化为稳定增益。

\subsection{类别级误差分析}

为回答"提升来自哪里"，我们进一步统计各方法在三类有效标签（LumA/LumB/Basal）上的 fold-level 平均召回率与 F1 分数。该表由 \texttt{outputs/logs/class\_error\_analysis.csv} 自动生成，避免手工录入误差。结果显示四种方法的最难类别均为 LumB，其中 Concat 的 LumB 召回率最高（0.8460），相较 RNA-only（0.8020）提升明显；MOFA 在该类别上的召回最低（0.7120），与其整体方差较大现象一致。从 F1 分数看，LumA 与 Basal 类的表现相对稳定，而 LumB 的类内差异最大，提示该类别仍有显著的分类难度。
