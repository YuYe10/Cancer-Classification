\section{方法}

\subsection{整体技术路线}

本文的实验流程可以概括为：数据读取、样本对齐、标签构建、训练测试划分、特征预处理、多组学融合、分类建模与结果评估。与许多仅在模型层面比较方法的工作不同，本文更强调流程一致性。因为在多组学任务中，预处理和样本映射往往比分类器本身更容易影响最终结果。

从实现角度看，当前仓库中的核心流程由 \texttt{src/pipeline.py} 统一编排。该流水线根据配置文件中的 \texttt{exp} 字段切换不同实验分支：\texttt{rna} 表示单组学基线，\texttt{concat} 表示特征拼接，\texttt{mofa} 表示潜在表示学习。与此同时，\texttt{ablation} 字段则用于控制是否使用 RNA、是否使用甲基化、是否启用高方差筛选，从而形成消融实验矩阵。

\subsection{RNA-only 基线}

RNA-only 实验是全文的最基本对照组。其目的并不是追求最优性能，而是作为多组学方法是否真正有效的参照。若多组学方法无法明显超过 RNA-only，则说明甲基化信息并未被有效利用，或者融合策略没有形成有效互补。

在当前实现中，RNA-only 的输入来自 \texttt{RNAPreprocessor} 处理后的表达矩阵。该处理器会先进行对数变换，再利用高方差特征筛选保留区分度较高的基因，最后在训练集上拟合标准化器。这样做的直接效果是：模型输入维度被压缩，数值尺度更稳定，且对 SVM 这类对特征尺度较敏感的方法更加友好。

\subsection{Concat 特征拼接}

Concat 是最直接的多组学融合方式。其核心思想是将 RNA 与甲基化特征在样本维度上对齐后沿特征维拼接，形成一个更宽的输入矩阵。与复杂的融合算法相比，Concat 几乎没有额外的模型学习负担，因此常被视为多组学研究中的第一层对照。

Concat 的优点在于简单、透明、实现成本低；缺点则在于它假设不同组学之间可以被直接并列处理，而没有显式建模跨组学相关性。在高维小样本场景下，这种直接拼接还可能放大冗余与噪声。因此，Concat 的实验结果通常用于判断“简单融合是否足够”，以及进一步复杂化模型是否真的值得。

\subsection{MOFA 潜在因子融合}

MOFA 是本文的核心多组学融合方法。它试图从多个组学中提取共享的低维因子，使这些因子同时保留跨组学一致性与组学特有信息。相比拼接，MOFA 不再把所有特征平铺到同一空间，而是尝试在潜变量层面寻找对样本分类更有帮助的结构。

从数学上看，MOFA 可视为一类矩阵分解与潜变量模型，其目标是在尽可能少的潜在因子下重构多个视图数据。对于分类任务而言，学习到的因子矩阵可作为新的低维输入，交给下游分类器使用。其优势包括：
\begin{itemize}
  \item 维度显著降低，减少过拟合风险；
  \item 提供较强的跨组学解释性；
  \item 更适合识别共享信号而非单一组学的局部噪声。
\end{itemize}

当前仓库通过 \texttt{mofapy2} 运行 MOFA，已经解决了输入数据结构、训练选项初始化以及训练/测试适配问题。需要注意的是，当前代码中 MOFA 接口尚未提供标准的 \texttt{transform} 形式，因此在工程上采用了合并样本一次性拟合并切回训练/测试的实现方式。该设计保证了实验可运行，但在论文中应明确说明其实现约束，避免读者误解为严格的独立测试评估流程。

\subsection{特征筛选与标准化}

高维特征筛选是多组学分类的基本步骤。本文采用方差筛选作为首选方法，原因在于它简单、高效，且与当前实验的目标一致：在不引入复杂先验的前提下尽可能保留具有区分度的特征。对于 RNA 和甲基化，Top-K 方差筛选分别在各自组学内部执行，以避免不同组学之间的方差信号被直接比较。

标准化则采用 z-score 形式，目的是消除不同特征之间的尺度差异。对于 SVM 这类算法而言，尺度不统一会直接影响核函数距离计算与决策边界学习。当前实现中，标准化器只在训练集上拟合，测试集仅作 transform，从而遵循基本的机器学习防泄露原则。

\subsection{分类器设计}

本文统一采用支持向量机作为分类器，以便将实验重点放在特征融合而非分类器复杂度上。SVM 的优势在于：
\begin{itemize}
  \item 适合中小样本、高维输入；
  \item 对边界样本有较强的判别能力；
  \item 与高方差筛选后的特征输入配合较好；
  \item 便于与其他模型做公平对比。
\end{itemize}

在当前代码中，SVM 默认开启概率估计，以便后续扩展 ROC-AUC、置信度分析或 SHAP 解释等任务。虽然本轮实验主要输出 Accuracy 与分类报告，但这种接口设计为后续扩展保留了空间。

\subsection{消融实验设计}

为了说明每个模块的实际贡献，本文设计了三组消融：
\begin{enumerate}
  \item 去甲基化：仅使用 RNA，分析甲基化对性能的增益；
  \item 去 RNA：仅使用甲基化，分析表达组学对性能的增益；
  \item 不做特征选择：保留原始组学输入，分析高方差筛选的作用。
\end{enumerate}

这三组消融分别对应“组学贡献”和“预处理贡献”两个维度。其中，前两组消融用于比较不同模态的互补程度，第三组消融则用于验证特征筛选是否确实能够缓解高维噪声问题。消融实验的结论不仅能说明模型是否有效，也能说明当前工程化流程中的哪些部分是必要的。

\subsection{评价指标}

当前实验主要采用分类准确率（Accuracy）与分类报告（Precision、Recall、F1-score、Support）作为指标。Accuracy 适合快速衡量整体分类效果，但当类别分布不均衡时，它容易掩盖少数类性能问题。因此，本文同时关注宏平均（macro avg）与加权平均（weighted avg）的表现。

在论文撰写中，建议进一步补充每个类别的召回率与 F1-score，以分析模型在 LumA、LumB、HER2、Basal 四个亚型上的差异。若后续扩展到多次随机种子实验，还可以增加均值与标准差，从而增强结论稳定性。
