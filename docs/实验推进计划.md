## 4. 每轮固定动作

### 4.1 实验执行

1. 跑主线配置并记录summary_v2.csv
2. 更新leaderboard（按accuracy_mean排序）
3. 对比上一轮的CI是否实质提升
4. 仅保留提升明显的配置进入下一轮

### 4.2 结果记录

1. **性能指标**：
   - Accuracy、Balanced Accuracy、Macro-F1（mean ± std）
   - 95% CI
   - Fold级详细结果
2. **统计检验**：
   - 成对t检验/置换检验
   - p值和效应量
3. **类别级分析**：
   - 混淆矩阵
   - 各类别precision/recall/F1
   - 错误样本列表
4. **可视化**：
   - 性能对比图（bar plot with error bars）
   - ROC/PR曲线
   - t-SNE/PCA降维可视化
   - 特征重要性图

### 4.3 论文更新

1. 更新实验结果表格
2. 补充统计分析
3. 更新讨论和结论
4. 检查逻辑链条一致性

### 4.4 进度检查机制

**定期检查时间节点**：
- 每周五：检查本周实验进展，更新leaderboard
- 每两周：对比CI变化，评估是否需要调整策略
- 每月：全面评估项目进度与目标的差距

**进度评估指标**：
1. **性能指标追踪**：
   - 各方法Accuracy均值是否稳定或提升
   - 95% CI是否收窄（表示结果更稳定）
   - 置换检验p值是否显著（< 0.05）
2. **任务完成度**：
   - 短期目标：全部完成（✅）
   - 中期目标：外部验证进度百分比
   - 论文结构完整性百分比
3. **代码与文档同步**：
   - 实验配置与论文描述一致性
   - 结果日志与图表生成一致性

**偏差识别与调整策略**：
| 偏差类型 | 识别标准 | 调整策略 |
|---------|---------|---------|
| 性能停滞 | 连续2周Accuracy提升<0.005 | 尝试新特征组合或调整模型 |
| CI未收窄 | 10轮后CI宽度无变化 | 增加CV repeats或检查数据质量 |
| 外部验证失败 | 2周内无法获取数据 | 改用内部时间切分验证 |
| 论文延迟 | 进度落后计划>1周 | 优先完成核心章节 |

**进度报告模板**：

## 9. 风险与应对

### 9.1 数据风险

**风险**：HER2类别样本不足，无法恢复四分类
**应对**：
- 合并多个数据源（TCGA + METABRIC）
- 采用数据增强（SMOTE、ADASYN）
- 或接受三分类，明确说明局限性

### 9.2 方法风险

**风险**：MOFA性能不稳定，无法达到预期效果
**应对**：
- 调整因子数量和likelihood函数
- 增加样本规模
- 或在讨论中说明其适用边界

### 9.3 统计风险

**风险**：样本量小，统计功效不足
**应对**：
- 增加重采样轮次（已完成：5x10）
- 使用bootstrap估计CI（已实现）
- 引入外部验证集（待开展）

### 9.4 时间风险

**风险**：实验周期长，无法按时完成
**应对**：
- 优先完成短期目标（已完成）
- 并行运行多个配置
- 合理终止条件
