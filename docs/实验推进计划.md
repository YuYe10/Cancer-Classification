## 2. 高分研究报告目标

### 2.1 核心目标

1. **科学性**：建立严格防泄露的统一评估框架，给出可复现的性能趋势
2. **完整性**：形成"问题-方法-结果-讨论-结论"完整证据链
3. **创新性**：在BRCA单癌种任务中实现可审计的严格评估，给出融合递进链条实证
4. **实用性**：提供可复现的代码框架和明确的改进路线

### 2.2 Novelty定位

本研究不宣称算法发明，而是聚焦"研究设计创新"：
- 在BRCA单癌种小样本条件下建立严格防泄露评估流程
- 构建并实证比较四层融合递进链条（RNA-only→Concat→MOFA→Stacking）
- 将类别级误差（尤其LumB易错）纳入主讨论，形成具体改进清单

## 3. 迭代路线图

### 3.1 短期目标（1-2周，仓库内可完成）

**目标**：提升统计功效和证据密度

**具体任务**：
1. **增加重采样轮次**：将CV从5x3提升至5x10或10x5
2. **补充多指标显著性检验**：Balanced Accuracy、Macro-F1的成对检验
3. **完善类别级误差分析**：生成混淆矩阵、错误样本列表、关键特征贡献
4. **优化Stacking超参数**：grid search base_model_type、meta_model_type、meta_cv_splits
5. **补充可视化**：ROC曲线、PR曲线、t-SNE降维可视化

**预期收益**：
- 统计功效提升，置信区间收窄
- 明确LumB易错原因和改进方向
- 提升报告说服力和完整性

**终止条件**：
- 95% CI宽度 < 0.05
- 至少一对方法在p<0.05水平显著

### 3.2 中期目标（2-4周，需额外数据）

**目标**：提升泛化可信度和生物学解释深度

**具体任务**：
1. **引入外部验证集**：
   - 从UCSC Xena下载其他BRCA队列（如METABRIC）
   - 或使用TCGA不同平台数据（如RNA-seq vs Microarray）
   - 进行跨平台验证
2. **恢复HER2类别**：
   - 合并多个数据源增加HER2样本
   - 或采用数据增强技术（SMOTE、ADASYN）
3. **强化生物学解释**：
   - 关键特征通路富集分析（KEGG、GO）
   - 文献证据映射（关键基因/位点与亚型关系）
   - 错误样本的临床特征分析
4. **消融实验扩展**：
   - 单独RNA vs 单独甲基化
   - 不同特征选择阈值
   - 不同标准化方法

**预期收益**：
- 缓解"仅单数据源有效"质疑
- 恢复完整四分类任务
- 形成"性能+机制"双重贡献

**终止条件**：
- 外部验证集上最佳方法Accuracy > 0.85
- HER2类别召回率 > 0.70

### 3.3 长期目标（1-2月，研究深化）

**目标**：探索更强融合方法和学术发表

**具体任务**：
1. **探索先进融合方法**：
   - 深度学习：Autoencoder、VAE、多模态Transformer
   - 图神经网络：基于PPI网络或基因调控网络
   - 注意力机制：模态间注意力、特征级注意力
2. **时间切分验证**：按样本采集时间划分训练/测试集
3. **可解释性增强**：
   - SHAP值分析
   - 因子重要性可视化
   - 决策边界可视化
4. **论文完善**：
   - 补充相关工作（多组学融合文献综述）
   - 扩展讨论（失败案例、边界条件、未来方向）
   - 完善参考文献

**预期收益**：
- 探索融合方法上限
- 形成可发表的完整研究
- 为后续研究提供基准

**终止条件**：
- 最佳方法Accuracy > 0.92
- 完成论文初稿

## 4. 每轮固定动作

### 4.1 实验执行

1. 跑主线配置并记录summary_v2.csv
2. 更新leaderboard（按accuracy_mean排序）
3. 对比上一轮的CI是否实质提升
4. 仅保留提升明显的配置进入下一轮

### 4.2 结果记录

1. **性能指标**：
   - Accuracy、Balanced Accuracy、Macro-F1（mean ± std）
   - 95% CI
   - Fold级详细结果

2. **统计检验**：
   - 成对t检验/置换检验
   - p值和效应量

3. **类别级分析**：
   - 混淆矩阵
   - 各类别precision/recall/F1
   - 错误样本列表

4. **可视化**：
   - 性能对比图（bar plot with error bars）
   - ROC/PR曲线
   - t-SNE/PCA降维可视化
   - 特征重要性图

### 4.3 论文更新

1. 更新实验结果表格
2. 补充统计分析
3. 更新讨论和结论
4. 检查逻辑链条一致性

## 9. 风险与应对

### 9.1 数据风险

**风险**：HER2类别样本不足，无法恢复四分类
**应对**：
- 合并多个数据源（TCGA + METABRIC）
- 采用数据增强（SMOTE、ADASYN）
- 或接受三分类，明确说明局限性

### 9.2 方法风险

**风险**：MOFA性能不稳定，无法达到预期效果
**应对**：
- 调整因子数量和likelihood函数
- 增加样本规模
- 或在讨论中说明其适用边界

### 9.3 统计风险

**风险**：样本量小，统计功效不足
**应对**：
- 增加重采样轮次
- 使用bootstrap估计CI
- 引入外部验证集

### 9.4 时间风险

**风险**：实验周期长，无法按时完成
**应对**：
- 优先完成短期目标
- 并行运行多个配置
- 合理终止条件
