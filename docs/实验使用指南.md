# 实验使用指南

本指南用于快速复现实验、运行消融对比，并说明常见问题。

## 1. 环境准备

在项目根目录执行：

```bash
pip install -r requirements.txt
```

本仓库默认使用内置解释器 `./medical/bin/python3`。

## 2. 数据与标签约定

- RNA 数据：`datasets/TCGA-BRCA.star_counts.csv`
- 甲基化数据：`datasets/TCGA-BRCA.methylation450.csv`
- 临床标签：`datasets/brca_pam50.csv`

临床文件要求：

- 分隔符为制表符（TSV）
- 必须包含 `sample` 列
- 标签列支持 `PAM50` 或 `label`
- 仅保留四个亚型：`LumA`、`LumB`、`HER2`、`Basal`

## 3. 主实验

### 3.1 一键运行全部主实验

```bash
bash run_all.sh
```

默认依次运行：

- `config/exp_rna.yaml`
- `config/exp_concat.yaml`
- `config/exp_mofa.yaml`
- `config/exp_stacking.yaml`

### 3.2 单独运行某个实验

```bash
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/exp_rna.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/exp_concat.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/exp_mofa.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/exp_stacking.yaml
```

### 3.3 严格交叉验证实验

如果你要看更可信的主结果，建议优先跑重复分层交叉验证：

```bash
bash scripts/run_strict_cv.sh
```

默认依次运行：

- RNA-only CV：`config/exp_rna_cv.yaml`
- Concat CV：`config/exp_concat_cv.yaml`
- Stacking CV：`config/exp_stacking.yaml`
- Concat + Random Forest CV：`config/exp_concat_rf_cv.yaml`

说明：

- 当前 `MOFA` 还没有严格无泄漏的 transform 路径，因此暂不纳入严格 CV 批处理
- 若需要比较 `MOFA`，建议保留在主实验 holdout 结果中，作为探索性对照

### 3.4 Stacking SOTA 批量实验

如果你要继续冲击更高 ACC，可运行更激进的 stacking 配置：

```bash
bash scripts/run_stacking_sota.sh
```

默认依次运行：

- XGBoost + XGBoost：`config/exp_stacking_sota_cv.yaml`
- Random Forest + XGBoost：`config/exp_stacking_hybrid_cv.yaml`
- Stable XGBoost + XGBoost：`config/exp_stacking_sota_stable_cv.yaml`

### 3.5 结果自动落盘

`experiments/run.py` 默认会自动写入：

- 详细结果 JSON：`outputs/logs/<config>_<tag>_<timestamp>.json`
- 汇总表 CSV：`outputs/logs/summary.csv`

可选参数：

- `--tag <name>`：为本次实验打标签（如 `cv_main`）
- `--output-dir <path>`：指定结果输出目录
- `--no-save`：仅打印结果，不落盘

### 3.6 结果排行榜汇总

对历史实验进行自动排序和导出：

```bash
PYTHONPATH=. ./medical/bin/python3 scripts/summarize_results.py --mode cv --metric accuracy_mean --top-k 20
```

如果你使用了带 CI 字段的新评估记录（写入 `summary_v2.csv`），请显式指定：

```bash
PYTHONPATH=. ./medical/bin/python3 scripts/summarize_results.py --summary-csv outputs/logs/summary_v2.csv --mode cv --metric accuracy_mean --top-k 20
```

默认输出：

- 排行榜 Markdown：`outputs/logs/leaderboard.md`
- 排行榜 CSV：`outputs/logs/leaderboard.csv`

也可以一键生成常用榜单（CV 总榜、SOTA 子榜、全量总榜）：

```bash
bash scripts/build_leaderboards.sh
```

该脚本还会额外产出：

- 最佳配置汇总：`outputs/logs/best_configs_by_exp.md`
- ACC 目标监控（>=0.95）：`outputs/logs/target_hits_095.md`

也可以单独运行：

```bash
PYTHONPATH=. ./medical/bin/python3 scripts/export_best_configs.py --group-by exp --metric accuracy_mean
PYTHONPATH=. ./medical/bin/python3 scripts/monitor_target.py --threshold 0.95
```

### 3.7 鲁棒评估与统计检验（推荐）

为避免“单次 holdout 同分”误导，建议运行鲁棒评估批处理：

```bash
bash scripts/run_robust_evaluation.sh
```

该脚本会自动：

- 运行 RNA / Concat / Concat+RF / Stacking 的 repeated CV
- 生成每个方法的均值、标准差与 95% CI（写入 JSON 与汇总 CSV）
- 对方法间做置换检验并输出统计报告：
  - `outputs/logs/statistical_evaluation.md`

也可手动指定若干 CV 结果 JSON 进行比较：

```bash
PYTHONPATH=. ./medical/bin/python3 scripts/statistical_evaluation.py \
  --files <cv_json_1> <cv_json_2> <cv_json_3> \
  --metric accuracy \
  --out-md outputs/logs/statistical_evaluation.md
```

### 3.8 从真实日志生成可视化图表

为避免“模拟数据图”，推荐直接基于 `summary.csv` 生成结果图：

```bash
PYTHONPATH=. ./medical/bin/python3 scripts/plot_experiment_results.py \
  --summary-csv outputs/logs/summary.csv \
  --out-dir outputs/figures \
  --threshold 0.95
```

该脚本会生成：

- `outputs/figures/real_holdout_metrics.png`（holdout 多指标对比）
- `outputs/figures/real_cv_metrics_errorbar.png`（CV 均值与标准差）
- `outputs/figures/real_score_timeline.png`（实验时间线）
- `outputs/figures/real_target_gap.png`（距离目标 ACC 的差距）
- `outputs/figures/real_results_latest_snapshot.csv`（每个实验模式的最新快照）

## 4. 消融实验

### 4.1 一键运行全部消融实验

```bash
bash scripts/run_ablation.sh
```

默认依次运行：

- 去甲基化（仅 RNA）：`config/ablation_no_meth.yaml`
- 去 RNA（仅甲基化）：`config/ablation_no_rna.yaml`
- 不做特征选择：`config/ablation_no_fs.yaml`

### 4.2 单独运行某个消融实验

```bash
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/ablation_no_meth.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/ablation_no_rna.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/ablation_no_fs.yaml
```

建议为消融批次加标签，便于后续筛选：

```bash
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/ablation_no_meth.yaml --tag ablation
```

## 5. 配置说明

关键字段如下：

- `exp`
  - `rna`：仅 RNA 特征
  - `meth`：仅甲基化特征
  - `concat`：RNA 与甲基化拼接
  - `mofa`：MOFA 潜在因子
  - `stacking`：RNA / Meth 双分支 late fusion + 元分类器

- `preprocess.rna_top_var` / `preprocess.meth_top_var`
  - 各组学保留的高方差特征数

- `model.test_size`
  - 测试集比例

- `model.random_state`
  - 随机种子

- `evaluation.mode`
  - `holdout`：单次分层划分
  - `repeated_cv`：重复分层交叉验证

- `evaluation.folds` / `evaluation.repeats`
  - CV 的折数与重复次数

- `ablation`
  - `use_rna`：是否启用 RNA
  - `use_meth`：是否启用甲基化
  - `use_feature_selection`：是否启用 Top-K 特征筛选

## 6. 当前实现中的防泄露策略

- 先切分训练/测试，再在训练集上拟合标准化参数
- 特征筛选集合先确定，再对训练/测试使用相同特征集合

说明：

- 由于当前 `run_mofa` 接口仅提供拟合+提取潜在因子，`mofa` 分支采用“合并样本拟合一次后再切回训练/测试”的策略以保证流程可运行。
- `stacking` 分支使用训练折上的 OOF 元特征训练元分类器，避免测试折信息泄漏到融合层。

## 7. 统计表格与对比生成

为论文集成新的 CI + p-value 结果表，可使用以下脚本：

### 7.1 生成对比表格

运行鲁棒评估后，从多个 CV JSON 文件生成 LaTeX / Markdown 表格：

```bash
PYTHONPATH=. ./medical/bin/python3 scripts/generate_comparison_tables.py \
  --files outputs/logs/exp_rna_cv_<tag>_*.json \
          outputs/logs/exp_concat_cv_<tag>_*.json \
          outputs/logs/exp_stacking_<tag>_*.json \
  --metrics accuracy balanced_accuracy macro_f1 weighted_f1 \
  --out-latex outputs/logs/method_comparison.tex \
  --out-md outputs/logs/method_comparison.md
```

输出：

- `method_comparison.tex`：可直接插入论文的 LaTeX 表格（带 95% CI）
- `method_comparison.md`：Markdown 版本（便于快速审查）
- `method_comparison_summary.csv`：原始数据 CSV

### 7.2 生成统计可视化图表

从 CV 结果生成出版级别的对比图表：

```bash
PYTHONPATH=. ./medical/bin/python3 scripts/generate_statistical_plots.py \
  --files outputs/logs/exp_rna_cv_<tag>_*.json \
          outputs/logs/exp_concat_cv_<tag>_*.json \
          outputs/logs/exp_stacking_<tag>_*.json \
  --metric accuracy \
  --out-dir outputs/figures
```

输出：

- `statistical_accuracy_ci.png`：误差条图（均值 + 95% CI）
- `statistical_accuracy_distribution.png`：箱线图（fold-level 分布）

### 7.3 常见的组合工作流

**场景 1：快速比较两个方法**

```bash
# 指定两个已有的 CV JSON 文件
PYTHONPATH=. ./medical/bin/python3 scripts/statistical_evaluation.py \
  --files outputs/logs/exp_rna_cv_*.json \
          outputs/logs/exp_stacking_*.json \
  --metric accuracy \
  --out-md outputs/logs/quick_comparison.md

cat outputs/logs/quick_comparison.md
```

**场景 2：生成论文完整结果表**

```bash
# 第一步：运行完整的鲁棒评估（耗时 30-60 分钟）
bash scripts/run_robust_evaluation.sh

# 第二步：生成 LaTeX 表格
PYTHONPATH=. ./medical/bin/python3 scripts/generate_comparison_tables.py \
  --files outputs/logs/exp_rna_cv_robust_eval_*.json \
          outputs/logs/exp_concat_cv_robust_eval_*.json \
          outputs/logs/exp_concat_rf_cv_robust_eval_*.json \
          outputs/logs/exp_stacking_robust_eval_*.json \
  --out-latex outputs/logs/paper_results_table.tex \
  --out-md outputs/logs/paper_results_table.md

# 第三步：手动将 paper_results_table.tex 内容复制到论文中
# 并根据 paper_results_table.md 的 p-value 信息更新 Discussion
```

## 8. 常见问题排查

### 8.1 `ModuleNotFoundError: No module named src`

请在项目根目录运行，并加上 `PYTHONPATH=.`。

### 8.2 临床标签相关错误（如 `Unknown clinical labels`）

检查 `datasets/brca_pam50.csv`：

- 是否包含 `sample` 和 `PAM50`/`label`
- 标签是否在 `LumA/LumB/HER2/Basal` 范围

### 8.3 MOFA 报错（维度/样本数相关）

优先使用默认配置。若修改 `mofa.factors`，请保证其不超过有效样本或特征维度上限。

## 9. 结果解读建议

建议按以下顺序比较：

1. 主实验 holdout：`rna` vs `concat` vs `mofa` vs `stacking`
2. 严格 CV：`rna_cv` vs `concat_cv` vs `stacking`
3. 消融实验：去甲基化、去 RNA、不做特征选择
4. 关注指标：Accuracy、balanced Accuracy、各类别 precision/recall/F1、macro avg

通常预期：

- 去掉关键组学后性能下降或类别不均衡加剧
- 去掉特征选择后泛化能力下降（指标波动明显）
