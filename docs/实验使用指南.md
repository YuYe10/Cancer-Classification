# 实验使用指南

本指南用于快速复现实验、运行消融对比，并说明常见问题。

## 1. 环境准备

在项目根目录执行：

```bash
pip install -r requirements.txt
```

本仓库默认使用内置解释器 `./medical/bin/python3`。

## 2. 数据与标签约定

- RNA 数据：`datasets/TCGA-BRCA.star_counts.csv`
- 甲基化数据：`datasets/TCGA-BRCA.methylation450.csv`
- 临床标签：`datasets/brca_pam50.csv`

临床文件要求：

- 分隔符为制表符（TSV）
- 必须包含 `sample` 列
- 标签列支持 `PAM50` 或 `label`
- 仅保留四个亚型：`LumA`、`LumB`、`HER2`、`Basal`

## 3. 主实验

### 3.1 一键运行全部主实验

```bash
bash run_all.sh
```

默认依次运行：

- `config/exp_rna.yaml`
- `config/exp_concat.yaml`
- `config/exp_mofa.yaml`
- `config/exp_stacking.yaml`

### 3.2 单独运行某个实验

```bash
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/exp_rna.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/exp_concat.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/exp_mofa.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/exp_stacking.yaml
```

### 3.3 严格交叉验证实验

如果你要看更可信的主结果，建议优先跑重复分层交叉验证：

```bash
bash scripts/run_strict_cv.sh
```

默认依次运行：

- RNA-only CV：`config/exp_rna_cv.yaml`
- Concat CV：`config/exp_concat_cv.yaml`
- Stacking CV：`config/exp_stacking.yaml`
- Concat + Random Forest CV：`config/exp_concat_rf_cv.yaml`

说明：

- 当前 `MOFA` 还没有严格无泄漏的 transform 路径，因此暂不纳入严格 CV 批处理
- 若需要比较 `MOFA`，建议保留在主实验 holdout 结果中，作为探索性对照

### 3.4 Stacking SOTA 批量实验

如果你要继续冲击更高 ACC，可运行更激进的 stacking 配置：

```bash
bash scripts/run_stacking_sota.sh
```

默认依次运行：

- XGBoost + XGBoost：`config/exp_stacking_sota_cv.yaml`
- Random Forest + XGBoost：`config/exp_stacking_hybrid_cv.yaml`
- Stable XGBoost + XGBoost：`config/exp_stacking_sota_stable_cv.yaml`

### 3.5 结果自动落盘

`experiments/run.py` 默认会自动写入：

- 详细结果 JSON：`outputs/logs/<config>_<tag>_<timestamp>.json`
- 汇总表 CSV：`outputs/logs/summary.csv`

可选参数：

- `--tag <name>`：为本次实验打标签（如 `cv_main`）
- `--output-dir <path>`：指定结果输出目录
- `--no-save`：仅打印结果，不落盘

### 3.6 结果排行榜汇总

对历史实验进行自动排序和导出：

```bash
PYTHONPATH=. ./medical/bin/python3 scripts/summarize_results.py --mode cv --metric accuracy_mean --top-k 20
```

默认输出：

- 排行榜 Markdown：`outputs/logs/leaderboard.md`
- 排行榜 CSV：`outputs/logs/leaderboard.csv`

也可以一键生成常用榜单（CV 总榜、SOTA 子榜、全量总榜）：

```bash
bash scripts/build_leaderboards.sh
```

该脚本还会额外产出：

- 最佳配置汇总：`outputs/logs/best_configs_by_exp.md`
- ACC 目标监控（>=0.95）：`outputs/logs/target_hits_095.md`

也可以单独运行：

```bash
PYTHONPATH=. ./medical/bin/python3 scripts/export_best_configs.py --group-by exp --metric accuracy_mean
PYTHONPATH=. ./medical/bin/python3 scripts/monitor_target.py --threshold 0.95
```

## 4. 消融实验

### 4.1 一键运行全部消融实验

```bash
bash scripts/run_ablation.sh
```

默认依次运行：

- 去甲基化（仅 RNA）：`config/ablation_no_meth.yaml`
- 去 RNA（仅甲基化）：`config/ablation_no_rna.yaml`
- 不做特征选择：`config/ablation_no_fs.yaml`

### 4.2 单独运行某个消融实验

```bash
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/ablation_no_meth.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/ablation_no_rna.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/ablation_no_fs.yaml
```

建议为消融批次加标签，便于后续筛选：

```bash
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/ablation_no_meth.yaml --tag ablation
```

## 5. 配置说明

关键字段如下：

- `exp`
  - `rna`：仅 RNA 特征
  - `meth`：仅甲基化特征
  - `concat`：RNA 与甲基化拼接
  - `mofa`：MOFA 潜在因子
  - `stacking`：RNA / Meth 双分支 late fusion + 元分类器

- `preprocess.rna_top_var` / `preprocess.meth_top_var`
  - 各组学保留的高方差特征数

- `model.test_size`
  - 测试集比例

- `model.random_state`
  - 随机种子

- `evaluation.mode`
  - `holdout`：单次分层划分
  - `repeated_cv`：重复分层交叉验证

- `evaluation.folds` / `evaluation.repeats`
  - CV 的折数与重复次数

- `ablation`
  - `use_rna`：是否启用 RNA
  - `use_meth`：是否启用甲基化
  - `use_feature_selection`：是否启用 Top-K 特征筛选

## 6. 当前实现中的防泄露策略

- 先切分训练/测试，再在训练集上拟合标准化参数
- 特征筛选集合先确定，再对训练/测试使用相同特征集合

说明：

- 由于当前 `run_mofa` 接口仅提供拟合+提取潜在因子，`mofa` 分支采用“合并样本拟合一次后再切回训练/测试”的策略以保证流程可运行。
- `stacking` 分支使用训练折上的 OOF 元特征训练元分类器，避免测试折信息泄漏到融合层。

## 7. 常见问题排查

### 7.1 `ModuleNotFoundError: No module named src`

请在项目根目录运行，并加上 `PYTHONPATH=.`。

### 7.2 临床标签相关错误（如 `Unknown clinical labels`）

检查 `datasets/brca_pam50.csv`：

- 是否包含 `sample` 和 `PAM50`/`label`
- 标签是否在 `LumA/LumB/HER2/Basal` 范围

### 7.3 MOFA 报错（维度/样本数相关）

优先使用默认配置。若修改 `mofa.factors`，请保证其不超过有效样本或特征维度上限。

## 8. 结果解读建议

建议按以下顺序比较：

1. 主实验 holdout：`rna` vs `concat` vs `mofa` vs `stacking`
2. 严格 CV：`rna_cv` vs `concat_cv` vs `stacking`
3. 消融实验：去甲基化、去 RNA、不做特征选择
4. 关注指标：Accuracy、balanced Accuracy、各类别 precision/recall/F1、macro avg

通常预期：

- 去掉关键组学后性能下降或类别不均衡加剧
- 去掉特征选择后泛化能力下降（指标波动明显）
