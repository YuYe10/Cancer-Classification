# 实验使用指南（主线精简版）

本指南仅覆盖论文主线实验：RNA-only、Concat、MOFA、Stacking。

## 1. 环境准备

在项目根目录执行：

```bash
pip install -r requirements.txt
```

默认解释器：

```bash
./medical/bin/python3
```

## 2. 数据约定

- RNA: datasets/TCGA-BRCA.star_counts.csv
- Methylation: datasets/TCGA-BRCA.methylation450.csv
- Clinical: datasets/brca_pam50.csv

标签列支持 PAM50 或 label，样本列为 sample。

## 3. 主线实验入口

### 3.1 一键运行主线

```bash
bash run_all.sh
```

默认顺序：

1. config/exp_rna_cv.yaml
2. config/exp_concat_cv.yaml
3. config/exp_mofa.yaml
4. config/exp_stacking.yaml

### 3.2 严格 CV 批处理

```bash
bash scripts/run_strict_cv.sh
```

说明：

- 主口径为 repeated stratified CV (5 folds x 3 repeats)
- MOFA 已支持训练折拟合 + 测试折投影，可纳入严格 CV 主比较

### 3.3 稳健评估与统计比较

```bash
bash scripts/run_robust_evaluation.sh
```

该脚本会：

1. 批量运行四方法主线
2. 生成最新结果集合
3. 调用统计比较脚本输出对比报告

### 3.4 单独运行某个配置

```bash
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/exp_rna_cv.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/exp_concat_cv.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/exp_mofa.yaml
PYTHONPATH=. ./medical/bin/python3 experiments/run.py --config config/exp_stacking.yaml
```

## 4. 输出结果说明

默认输出目录：outputs/logs

每次运行产生：

- 详细 JSON: <config>_<tag>_<timestamp>.json
- 汇总 CSV: summary.csv 或 summary_v2.csv

重点字段：

- accuracy / balanced_accuracy / macro_f1
- accuracy_mean / accuracy_ci95_low / accuracy_ci95_high
- fold_metrics（CV 时）

## 5. 排行榜与汇总

```bash
PYTHONPATH=. ./medical/bin/python3 scripts/summarize_results.py --summary-csv outputs/logs/summary_v2.csv --mode cv --metric accuracy_mean --top-k 20
```

输出：

- outputs/logs/leaderboard.md
- outputs/logs/leaderboard.csv

## 6. 内部强化（不进论文主结果）

如需继续冲击更高准确率，可运行 stacking 强化批处理：

```bash
bash scripts/run_stacking_sota.sh
```

该部分用于内部调参与迭代，不建议直接作为论文主结果表。

## 7. 常见问题

### 7.1 某一类别被自动剔除

当某类别样本数 < 2 时，会在训练前自动剔除并记录 dropped_labels。

### 7.2 summary.csv 与 summary_v2.csv 同时存在

这是兼容旧字段与新字段造成的正常现象。建议后续统一使用 summary_v2.csv。

### 7.3 如何查看“是否接近 90%+”

参考 docs/实验推进计划.md，按轮次比较 accuracy_mean 与 95% CI。
