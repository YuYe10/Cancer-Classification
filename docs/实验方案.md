# 🧬 Multi-Omics Subtype Classification 实验方案（完整版）

---

## 1️⃣ 研究目标（What & Why）

### 🎯 目标

在**单一癌种（TCGA-BRCA）内部进行分子亚型分类**，构建一个高性能、可解释的分类模型。

### ❓ 为什么做这个任务

* 分子亚型（如 Luminal A / Basal）直接影响治疗策略
* 单组学数据存在信息缺失
* 多组学融合可提升分类能力与生物解释性

---

## 2️⃣ 整体技术路线

```text
数据获取 → 数据预处理 → 特征选择 → 多组学融合 → 分类建模 → 评估 → 可解释性分析
```

---

## 3️⃣ 数据获取

### 📦 数据来源

* TCGA-BRCA（通过 UCSC Xena 下载）

### 📊 数据类型

| 数据              | 内容        | 作用     |
| --------------- | --------- | ------ |
| RNA-seq         | 基因表达（TPM） | 表达层面   |
| DNA Methylation | CpG甲基化水平  | 表观遗传层面 |
| Clinical        | 亚型标签      | 监督学习   |

---

### ❓ 为什么选这三类数据

* RNA反映**功能表达**
* 甲基化反映**调控机制**
* 两者互补，适合多组学建模

---

## 4️⃣ 数据预处理

---

### 4.1 样本对齐

#### ✔ 操作

```text
取 RNA / Meth / Clinical 三者样本交集
```

#### ❓ 为什么

* 多组学必须保证“同一个样本”
* 否则模型学习无意义

---

### 4.2 RNA处理

#### ✔ 操作

* log2(TPM + 1)
* 去除低表达基因
* 保留高方差基因（Top 2000）

#### ❓ 为什么

| 操作    | 原因        |
| ----- | --------- |
| log变换 | 减少极端值影响   |
| 去低表达  | 降噪        |
| 高方差筛选 | 保留有区分力的特征 |

---

### 4.3 甲基化处理

#### ✔ 操作

* 删除缺失值
* 选Top 2000高方差probe

#### ❓ 为什么

* 甲基化维度极高（30万+）
* 必须降维，否则过拟合严重

---

### 4.4 标准化

#### ✔ 操作

```text
z-score标准化（均值0 方差1）
```

#### ❓ 为什么

* 不同组学尺度不同
* 防止某一组学主导模型

---

## 5️⃣ 标签构建

#### ✔ 操作

将亚型映射为数字标签：

```text
LumA → 0
LumB → 1
HER2 → 2
Basal → 3
```

#### ❓ 为什么

* 机器学习模型只能处理数值标签

---

## 6️⃣ 特征选择（Feature Selection）

---

### ✔ 方法

* 方差筛选（Top 2000）

### ❓ 为什么

* 高维数据（p >> n）
* 降低过拟合风险
* 提升训练稳定性

---

## 7️⃣ 多组学融合

---

### 🔵 方法1：RNA-only（Baseline）

#### ✔ 操作

仅使用RNA数据训练模型

#### ❓ 为什么

* 作为基准（baseline）
* 判断多组学是否有效

---

### 🟡 方法2：Concat（拼接）

#### ✔ 操作

```text
X = [RNA | Meth]
```

#### ❓ 为什么

* 最简单的融合方式
* 用于对比高级方法

---

### 🔴 方法3：MOFA（核心）

#### ✔ 操作

* 输入：RNA + Meth
* 输出：低维latent factors

#### ❓ 为什么

| 优势  | 解释         |
| --- | ---------- |
| 降维  | 减少噪声       |
| 融合  | 学习跨组学关系    |
| 可解释 | 潜在因子具有生物意义 |

### 🟢 方法4：Stacking / Late Fusion

#### ✔ 操作

* RNA 和 Meth 分别训练基模型
* 使用训练折上的 out-of-fold 概率作为元特征
* 再训练一个元分类器输出最终预测

#### ❓ 为什么

| 优势  | 解释 |
| --- | --- |
| 融合更强 | 比简单拼接更能利用不同模态的判别边界 |
| 更稳健 | 元层只看 OOF 特征，避免泄漏 |
| 易扩展 | 可替换基模型为 XGBoost / SVM / RF |

---

## 8️⃣ 分类模型

---

### ✔ 使用模型

* SVM
* Random Forest（推荐）

---

### ❓ 为什么选择这些模型

| 模型  | 优势         |
| --- | ---------- |
| SVM | 适合高维小样本    |
| RF  | 抗过拟合 + 可解释 |

---

## 9️⃣ 训练与测试划分

#### ✔ 操作

```text
Train/Test = 80% / 20%
```

#### ❓ 为什么

* 评估模型泛化能力
* 避免过拟合

### 9.1 严格交叉验证

#### ✔ 操作

* 使用 repeated stratified k-fold 作为主结果
* 所有特征选择、标准化、融合层都在训练折内拟合

#### ❓ 为什么

* 单次划分波动较大
* 多组学高维数据更容易受偶然分割影响
* 更适合比较 SOTA pipeline

---

## 🔟 评价指标

---

### ✔ 分类任务

| 指标        | 作用    |
| --------- | ----- |
| Accuracy  | 整体准确率 |
| Precision | 精确率   |
| Recall    | 召回率   |
| F1-score  | 综合指标  |
| ROC-AUC   | 分类能力  |

---

### ❓ 为什么要多指标

* Accuracy 不全面
* AUC更稳定

### 10.1 推荐汇报格式

* `Accuracy mean ± std`
* `Balanced Accuracy mean ± std`
* `Macro-F1 mean ± std`
* `Confusion matrix`

## 11️⃣ 消融设计建议

### 11.1 原始三方法消融

* RNA-only vs RNA + Meth 拼接 vs MOFA
* 去甲基化 / 去 RNA / 不做特征选择

### 11.2 新增增强方法消融

* Stacking vs Concat
* XGBoost vs SVM
* 有无 OOF 元特征

### 11.3 评价原则

* 主结果优先报告严格 CV
* Holdout 结果作为补充展示
* MOFA 若无严格 transform，不与严格 CV 结果直接对比
